## Architecture ‚Äî Two-Agent Research Pipeline

KYP uses a two-agent research architecture. Both agents call Gemini, but they serve different purposes and run at different times.

### Agent 1: FlashResearcher (built, live)

Fast enrichment. Triggers on every user search.

**Flow:** User searches "kuru" ‚Üí FlashResearcher calls Gemini ‚Üí merges result into Firebase protocol record ‚Üí frontend navigates to Explore. If protocol has no score yet, fires DeepResearcher in background (fire-and-forget).

**What it fills:**
- summary, category, subcategory, chain
- links (project, twitter, discord)
- contracts[] (contract name + address array)
- deployed_date, age_summary
- forensics (admin_function_notes, deployer_wallet_age)
- founder_history (details, confidence_note)
- image (via Google favicon service)

**What it does NOT fill:**
- score / score_max (DeepResearcher handles this)
- risks, funding, use_cases, team (DeepResearcher handles these)
- who_its_for / who_its_not_for (DeepResearcher handles these)
- plain_summary (DeepResearcher handles this)

**Guarantees:**
- Canonical protocol schema via `normalizeProtocol()` ‚Äî every protocol in Firebase has the same shape
- Contract addresses live in `contracts[]` array (sole source of truth)
- Never overwrites existing Firebase data with empty values (deepMerge + isMeaningful)
- Skips protocols that already have a real image (already researched)
- Logs every field change for debugging
- Shared merge utilities extracted to `services/merge.js`

### Agent 2: DeepResearcher (built, live)

Deep analysis. Triggers automatically after FlashResearcher enrichment, or on-demand via API.

**Flow:** FlashResearcher finishes enrichment ‚Üí checks if `deep_research_status === "pending"` and `score === null` ‚Üí fires `deepResearch(id)` as detached async call ‚Üí returns immediately to user.

**What it fills:**
- **plain_summary** ‚Äî human-readable protocol description
- **score / score_max** ‚Äî deterministic scoring from Gemini evidence (8 weighted signals)
- **risks** (contract, community, structural) ‚Äî risk analysis from Gemini
- **funding** (has_funding_info, investors, source_note)
- **use_cases** ‚Äî extracted from protocol analysis
- **who_its_for / who_its_not_for** ‚Äî user persona analysis
- **team** ‚Äî team member research
- **restricted_jurisdictions** ‚Äî regulatory awareness
- **contract_verified** ‚Äî on-chain verification status

**Concurrency:** Firebase atomic lock ‚Äî sets `deep_research_status = "running"` before starting. If already `running`, skips.

**Failure:** Sets `status = "failed"` + `deep_research_error`. No auto-retry. Retries on next search if user searches the same protocol again.

**Scoring:** Gemini returns `evidence` object. Backend computes score deterministically from 8 signals:
| Signal | Weight | Condition |
|---|---|---|
| Audit | +10 | Known auditor |
| Team verified | +8 | Public team |
| Contract verified | +8 | On-chain verified |
| Funding | +6 | Has funding info |
| Community | +3-6 | Large/medium |
| Admin risk | +3-6 | Low/medium |
| Liquidity | +2-4 | Deep/moderate |
| Age | +2-4 | >60 or >180 days |
| Concentration | -2 to +4 | Holder distribution |

Score range: 0‚Äì50, mapped to 5 tiers (Rug potential / High risk / Caution / Solid / Secured).

**Shared utilities:** `deepMerge`, `isMeaningful`, `findChanges` extracted to `services/merge.js` ‚Äî used by both agents.

### Why two agents instead of one?

1. **User experience** ‚Äî Search is instant (< 5s). Deep research takes time. Don't block the search flow.
2. **Cost** ‚Äî Flash is cheap (single Gemini call). Deep is expensive (analysis + scoring).
3. **Reliability** ‚Äî Flash failing doesn't break search. Deep failing doesn't break the app.
4. **Progressive enhancement** ‚Äî Protocol works with just Flash. Deep makes it better.
